## 2. Overview
Отстраняването на грешки (debugging) и тестването са фундаментални дейности в софтуерното инженерство, които гарантират, че системите работят коректно и надеждно. Debugging е процесът на локализиране и коригиране на дефекти в кода, докато тестването е систематичното изпълнение на код с цел откриване на грешки и проверка на функционалността спрямо изискванията.

В контекста на изкуствения интелект, където системите често са сложни и непредсказуеми, тестването и отстраняването на грешки стават още по-важни. Те помагат да се гарантира, че моделите и алгоритмите не само работят технически, но и дават надеждни и обосновани резултати.

Тези процеси са интегрирани в жизнения цикъл на софтуерното развитие и често се повтарят итеративно. Добре структурираното тестване и ефективното отстраняване на грешки намаляват риска от критични проблеми в продукция и подобряват качеството на крайния продукт.

## 3. Key Concepts
- **Bug (Грешка)** – нежелано поведение или дефект в софтуера, който причинява неправилна работа. Можем да го сравним с „счупена част“ в машина, която пречи на нормалната ѝ функция.
- **Debugging (Отстраняване на грешки)** – процесът на идентифициране, локализиране и коригиране на грешки в кода. Аналогично на детектив, който търси причината за проблем.
- **Testing (Тестване)** – систематично изпълнение на код с цел откриване на грешки и проверка на функционалността. Можем да го разглеждаме като „контролна проверка“ преди пускане в експлоатация.
- **Unit Testing (Модулно тестване)** – тестване на отделни компоненти или функции поотделно, за да се гарантира тяхната коректност.
- **Integration Testing (Интеграционно тестване)** – проверка дали различните модули работят заедно правилно.
- **Regression Testing (Регресионно тестване)** – повторно тестване след промени в кода, за да се уверим, че новите промени не са въвели нови грешки.
- **Test Coverage (Покритие на тестовете)** – мярка за това колко голяма част от кода е тествана, подобно на „карта“, която показва колко територия е проверена.
- **Automated Testing (Автоматизирано тестване)** – използване на инструменти и скриптове за изпълнение на тестове без човешка намеса.

## 4. Step-by-step Learning Path
1. **Фокус:** Разбиране на основите на грешките и техните видове.  
   **Практическа задача:** Намерете и опишете 3 различни вида грешки в малък кодов фрагмент.  
   **Въпроси за припомняне:** Какво е bug? Какви са основните типове грешки?

2. **Фокус:** Научете се да използвате дебъгер (debugger).  
   **Практическа задача:** Стартирайте дебъгер върху прост Python скрипт и проследете изпълнението стъпка по стъпка.  
   **Въпроси за припомняне:** Какво е breakpoint? Какво означава step-over?

3. **Фокус:** Писане на модулни тестове.  
   **Практическа задача:** Напишете unit тестове за функция, която изчислява факториел.  
   **Въпроси за припомняне:** Какво е unit тест? Защо е важно да тестваме отделни функции?

4. **Фокус:** Автоматизиране на тестовете и интеграционно тестване.  
   **Практическа задача:** Настройте автоматизиран тестов скрипт, който тества няколко функции заедно.  
   **Въпроси за припомняне:** Какво е интеграционно тестване? Какви са предимствата на автоматизираното тестване?

5. **Фокус:** Практики за регресионно тестване и поддържане на тестовете.  
   **Практическа задача:** След промяна в кода, изпълнете регресионни тестове и анализирайте резултатите.  
   **Въпроси за припомняне:** Какво е регресионно тестване? Защо е важно да поддържаме тестовете актуални?

## 5. Examples
### Пример 1: Отстраняване на грешка с Python дебъгер
```python
def divide(a, b):
    return a / b

result = divide(10, 0)  # Тук възниква грешка ZeroDivisionError
```
Използвайте дебъгер, за да проследите къде точно възниква грешката и защо.

### Пример 2: Модулен тест с `unittest` в Python
```python
import unittest

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

class TestFactorial(unittest.TestCase):
    def test_factorial(self):
        self.assertEqual(factorial(5), 120)
        self.assertEqual(factorial(0), 1)

if __name__ == '__main__':
    unittest.main()
```

### Пример 3: Автоматизирано интеграционно тестване с pytest
```python
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

def test_operations():
    assert add(2, 3) == 5
    assert multiply(2, 3) == 6
```

## 6. Common Pitfalls
- Игнориране на малки грешки, които водят до по-големи проблеми по-късно. Винаги анализирайте и коригирайте грешките своевременно.
- Писане на тестове, които не покриват всички възможни случаи (лошо тестово покритие). Използвайте техники за измерване на покритието.
- Прекалено сложни тестове, които са трудни за поддръжка. Пишете ясни и прости тестове.
- Пренебрегване на регресионното тестване след промени в кода. Винаги изпълнявайте регресионни тестове, за да избегнете въвеждане на нови грешки.
- Ръчно тестване без автоматизация, което е бавно и податливо на човешка грешка. Автоматизирайте тестовете, когато е възможно.

## 8. Quick Recap
- Debugging е процес на локализиране и коригиране на грешки в софтуера.
- Тестването гарантира, че софтуерът работи според изискванията и открива дефекти.
- Unit тестовете проверяват отделни компоненти, а интеграционните – взаимодействието между тях.
- Регресионното тестване защитава срещу въвеждане на нови грешки след промени.
- Автоматизацията на тестовете повишава ефективността и надеждността.
- Добре структурираните тестове и дебъгинг намаляват риска от критични проблеми в продукция.
- Използването на дебъгер е ключов инструмент за разбиране и отстраняване на грешки.
