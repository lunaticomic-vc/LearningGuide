## 2. Overview
Bubble Sort, Selection Sort и Insertion Sort са три фундаментални алгоритъма за сортиране, които често се изучават като основа за разбиране на по-сложни техники. Те са сравнително прости за имплементация и визуализация, което ги прави отлична отправна точка за изучаване на алгоритми и анализ на сложност.

В контекста на AI и софтуерното инженерство, сортирането е ключова операция, която често се използва за организиране на данни, оптимизиране на търсения и подготовка на входни данни за по-сложни алгоритми. Разбирането на тези базови методи помага да се оцени кога и защо да се използват по-ефективни алгоритми.

Въпреки че тези алгоритми не са най-бързите за големи обеми данни, те демонстрират важни концепции като сравнения, размени и итерации, които са в основата на много други техники. Освен това, те са полезни за малки или почти сортирани списъци, където могат да бъдат изненадващо ефективни.

## 3. Key Concepts
- **Bubble Sort** – Алгоритъм, който многократно преминава през списъка, сравнява съседни елементи и ги разменя, ако са в неправилен ред. Мислете за мехурчета, които „изплуват“ до върха на списъка.
- **Selection Sort** – Алгоритъм, който избира най-малкия (или най-големия) елемент от неотсортированата част и го поставя на правилната позиция. Представете си, че избирате най-малкия плод от кошница и го поставяте в ред.
- **Insertion Sort** – Алгоритъм, който изгражда сортиран списък, като взема елемент по елемент и го вмъква на правилната позиция. Подобно на подреждането на карти в ръка.
- **Time Complexity** – Мярка за броя операции, необходими за сортиране, обикновено изразена в O(n²) за тези алгоритми.
- **In-place Sorting** – Сортиране, което не изисква допълнителна памет, а променя оригиналния списък.
- **Stable Sorting** – Запазване на относителния ред на равни елементи, важно при сортиране на сложни структури.

## 5. Examples

**Bubble Sort на Python:**
```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

print(bubble_sort([5, 1, 4, 2, 8]))
```

**Selection Sort на Python:**
```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i+1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

print(selection_sort([64, 25, 12, 22, 11]))
```

**Insertion Sort на Python:**
```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >=0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr

print(insertion_sort([12, 11, 13, 5, 6]))
```

## 6. Common Pitfalls
- **Прекомерно използване на Bubble Sort при големи данни** – алгоритъмът е бавен (O(n²)) и не е подходящ за големи масиви.
- **Забравяне за стабилност** – Selection Sort не е стабилен по подразбиране, което може да доведе до загуба на относителния ред при равни елементи.
- **Неправилно управление на индекси** – особено при Insertion Sort, грешки в индексирането могат да доведат до извънредни грешки или неправилно сортиране.
- **Игнориране на почти сортирани списъци** – Insertion Sort е много ефективен при такива случаи, но често се пренебрегва.
- **Липса на тестване с различни входни данни** – важно е да се тества с различни случаи (ред, обратен ред, случайни данни).

## 8. Quick Recap
- Bubble, Selection и Insertion Sort са базови алгоритми за сортиране с O(n²) сложност.  
- Bubble Sort сравнява и разменя съседни елементи, „изплувайки“ най-големия елемент.  
- Selection Sort избира най-малкия елемент и го поставя на правилното място.  
- Insertion Sort вмъква елементите в сортираната част, ефективен при почти сортирани данни.  
- Insertion Sort и Bubble Sort са стабилни, Selection Sort – не по подразбиране.  
- Всички са in-place алгоритми, не изискват допълнителна памет.  
- Изборът на алгоритъм зависи от размера и характеристиките на данните.
