## 2. Overview
В софтуерното инженерство и изкуствения интелект, "структура" се отнася до начина, по който данните са организирани и съхранявани, докато "операции" са действията, които се извършват върху тези данни. Тези две понятия са фундаментални за изграждането на ефективни и мащабируеми системи.

Структурите на данни осигуряват рамката, която позволява бърз достъп, модификация и управление на информацията. Операциите, от своя страна, дефинират как тази информация се използва, трансформира или анализира. В контекста на AI, правилният избор на структура и операции може да подобри значително производителността и точността на моделите.

Разбирането на структурата и операциите е ключово за оптимизиране на алгоритми, намаляване на сложността и избягване на излишно натоварване на системата. Това знание е основа за разработване на надеждни и ефективни софтуерни решения.

## 3. Key Concepts
- **Data Structure** – Организиран начин за съхранение и управление на данни, който позволява ефективен достъп и модификация. Можете да си го представите като различни видове "кутии" или "органайзери" за информация.
- **Operation** – Действие, което се извършва върху структурата на данните, като добавяне, изтриване, търсене или сортиране. Аналогично на действията, които извършвате върху съдържанието на една кутия.
- **Algorithm Complexity** – Мярка за това колко ресурси (време, памет) изисква дадена операция върху структурата. Представете си го като времето, необходимо да намерите книга в библиотека.
- **Abstract Data Type (ADT)** – Модел, който дефинира поведението на структурата чрез операции, без да се уточнява конкретната реализация. Подобно на рецепта, която описва какво трябва да се направи, без да казва как точно.
- **Mutable vs Immutable Structures** – Структури, които могат или не могат да бъдат променяни след създаване. Като бележник, в който може да пишете (mutable) срещу отпечатан документ (immutable).

3. **Фокус:** Дървовидни структури и графи  
   **Задача:** Създайте бинарно дърво и реализирайте обход in-order.  
   **Въпроси:** Какво е предимството на дървовидните структури? Какво представлява обходът in-order?

4. **Фокус:** Абстрактни типове данни и интерфейси  
   **Задача:** Дефинирайте интерфейс за списък и реализирайте два варианта: масивен и свързан списък.  
   **Въпроси:** Какво е предимството на използването на ADT? Как интерфейсите улесняват разширяемостта?

5. **Фокус:** Оптимизация на операции и анализ на сложност  
   **Задача:** Анализирайте времевата сложност на операциите върху вашите структури.  
   **Въпроси:** Какво означава O(n) и O(log n)? Как изборът на структура влияе на производителността?

## 5. Examples
### Пример 1: Имплементация на стек на Python
```python
class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        raise IndexError("pop from empty stack")

    def is_empty(self):
        return len(self.items) == 0
```

### Пример 2: Обход на бинарно дърво in-order на Python
```python
class Node:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

def inorder_traversal(root):
    if root:
        inorder_traversal(root.left)
        print(root.value)
        inorder_traversal(root.right)
```

### Пример 3: Линейно търсене в списък
```python
def linear_search(arr, target):
    for i, value in enumerate(arr):
        if value == target:
            return i
    return -1
```

## 6. Common Pitfalls
- **Недоразбиране на сложността на операциите:** Използване на неподходяща структура, което води до бавни операции. Решение: винаги анализирайте времевата и пространствената сложност.
- **Прекомерна мутация на структури:** Промяна на данни без контрол, което води до трудно откриваеми грешки. Решение: използвайте immutable структури, когато е възможно.
- **Игнориране на абстракцията:** Пряка работа с конкретни реализации вместо чрез интерфейси, което затруднява поддръжката и разширяването. Решение: дефинирайте и използвайте ADT.
- **Липса на тестване на операции:** Неизпълнение на достатъчно тестове за гранични случаи и нестандартни входни данни. Решение: пишете автоматизирани тестове за всички операции.

## 8. Quick Recap
- Структурите на данни организират информацията за ефективен достъп и модификация.
- Операциите са действията, които се извършват върху тези структури.
- Изборът на структура и операции влияе директно върху производителността на системата.
- Абстрактните типове данни дефинират поведение без конкретна реализация.
- Анализът на сложността помага да се изберат оптимални решения.
- Използването на интерфейси и immutable структури подобрява надеждността и разширяемостта.
- Практическото имплементиране и тестване са ключови за разбиране и усвояване.
