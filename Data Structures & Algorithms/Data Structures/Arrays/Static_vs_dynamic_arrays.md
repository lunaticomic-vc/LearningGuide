## 2. Overview
Масивите са фундаментална структура от данни, която позволява съхранение на множество елементи от един и същ тип в последователна памет. Те са широко използвани в софтуерното инженерство и изкуствения интелект за ефективна организация и достъп до данни.

Статичните масиви имат фиксиран размер, определен при създаването им, което ги прави бързи и предвидими по отношение на използването на паметта. Въпреки това, тяхната неподвижност ограничава гъвкавостта при работа с променливи обеми данни.

Динамичните масиви, от друга страна, могат да променят размера си по време на изпълнение, което ги прави подходящи за приложения, където обемът на данните не е предварително известен или постоянно се променя. Те използват механизми като преоразмеряване и копиране, за да осигурят тази гъвкавост, но с цената на по-голяма сложност и потенциално по-ниска производителност.

Разбирането на разликите между статични и динамични масиви е критично за оптималния дизайн на алгоритми и системи, особено в контекста на AI, където ефективността на обработка и управлението на паметта са ключови.

## 3. Key Concepts
- **Static Array (Статичен масив)** – Масив с фиксиран размер, зададен при създаване. Представете си го като шкаф с фиксиран брой чекмеджета, където не можете да добавите повече, отколкото има налични места.
- **Dynamic Array (Динамичен масив)** – Масив, който може да променя размера си по време на изпълнение. Аналогията е разтегателен шкаф, който може да добавя нови чекмеджета при нужда.
- **Capacity (Капацитет)** – Максималният брой елементи, които масивът може да съхранява без преоразмеряване.
- **Size (Размер)** – Текущият брой елементи в масива.
- **Reallocation (Преоразмеряване)** – Процесът на създаване на нов масив с по-голям капацитет и копиране на съществуващите елементи в него.
- **Amortized Cost (Амортизирана цена)** – Средната цена на операцията по добавяне в динамичен масив, като се вземат предвид и по-скъпите операции по преоразмеряване.
- **Contiguous Memory (Непрекъсната памет)** – Масивите съхраняват елементите си в последователни адреси в паметта, което позволява бърз достъп чрез индексиране.

## 5. Examples
### Пример 1: Статичен масив на C++
```cpp
int staticArray[5] = {1, 2, 3, 4, 5};
// Опит за добавяне на 6-ти елемент е невъзможен без преоразмеряване
```

### Пример 2: Динамичен масив на Python (списък)
```python
dynamic_array = []
for i in range(10):
    dynamic_array.append(i)
print(dynamic_array)  # [0, 1, 2, ..., 9]
```

### Пример 3: Имплементация на динамичен масив с преоразмеряване на C++
```cpp
class DynamicArray {
    int* data;
    int capacity;
    int size;
public:
    DynamicArray() : capacity(2), size(0) {
        data = new int[capacity];
    }
    void add(int value) {
        if (size == capacity) {
            capacity *= 2;
            int* newData = new int[capacity];
            for (int i = 0; i < size; i++)
                newData[i] = data[i];
            delete[] data;
            data = newData;
        }
        data[size++] = value;
    }
    int getSize() { return size; }
};
```

## 6. Common Pitfalls
- **Опит за добавяне на елементи в статичен масив извън капацитета му** – води до препълване на паметта или грешки. Използвайте динамичен масив или предварително планирайте размерите.
- **Често преоразмеряване на динамичен масив с малък прираст** – например увеличаване с 1 елемент, което води до лоша производителност. Използвайте експоненциално нарастване (удвояване).
- **Забравяне за освобождаване на памет при ръчно управление (например в C++)** – води до изтичане на памет.
- **Неправилно индексиране при копиране на елементи по време на преоразмеряване** – може да доведе до загуба или повреда на данни.
- **Игнориране на амортизираната сложност** – води до грешни предположения за производителността на динамичните масиви.

## 8. Quick Recap
- Статичните масиви имат фиксиран размер, а динамичните могат да се преоразмеряват.
- Динамичните масиви използват преоразмеряване и копиране за гъвкавост.
- Амортизираната цена на добавяне в динамичен масив е ефективна, въпреки скъпите операции по преоразмеряване.
- Статичните масиви са по-бързи и по-ефективни при фиксирани обеми данни.
- Динамичните масиви са незаменими при променливи и непредвидими обеми на данни.
- В AI системите изборът между двата типа масиви влияе върху производителността и управлението на паметта.
- Правилното управление на паметта е критично при работа с динамични масиви.
