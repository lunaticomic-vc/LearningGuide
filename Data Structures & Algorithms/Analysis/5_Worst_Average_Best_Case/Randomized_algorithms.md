## 2. Overview
Случайните алгоритми са алгоритми, които използват случайни числа или случайни решения като част от своята логика. Те се различават от детерминираните алгоритми по това, че могат да дават различни резултати при различни изпълнения за едни и същи входни данни. Това добавя елемент на вероятност към анализа и поведението им.

Тези алгоритми намират широко приложение в области като оптимизация, криптография, обработка на големи данни и машинно обучение. В контекста на изкуствения интелект, те често се използват за намиране на решения в сложни пространства, където детерминираните методи са твърде бавни или неефективни.

Основната причина за използването на случайни алгоритми е, че те могат да предложат по-добро средно време за изпълнение, по-малка сложност или по-лесна имплементация в сравнение с класическите методи. Въпреки това, те изискват разбиране на вероятностните свойства и оценка на риска от грешки.

## 3. Key Concepts
- **Randomized Algorithm** – алгоритъм, който използва случайни числа, за да влияе на своето поведение и резултат. Може да се разглежда като "решаващ с монета" в определени стъпки.
- **Monte Carlo Algorithm** – тип случайни алгоритми, които винаги завършват, но резултатът им е вероятностно коректен (с малка вероятност за грешка).
- **Las Vegas Algorithm** – случайни алгоритми, които винаги дават правилен резултат, но времето за изпълнение е случайно.
- **Expected Runtime** – средното време за изпълнение на алгоритъма, взето по вероятностното разпределение на случайните решения.
- **Probabilistic Analysis** – метод за анализ на алгоритми, който използва вероятностни модели, за да предскаже поведението им.
- **Random Seed** – начална стойност, която определя последователността на случайните числа, използвани от алгоритъма, позволявайки възпроизводимост.

## 5. Examples

### Пример 1: Тест на Милер-Рабин за простота (Monte Carlo)
```python
import random

def miller_rabin(n, k=5):
    if n < 2:
        return False
    # Намерете d и r, такива че n-1 = d * 2^r
    d = n - 1
    r = 0
    while d % 2 == 0:
        d //= 2
        r += 1

    for _ in range(k):
        a = random.randint(2, n - 2)
        x = pow(a, d, n)
        if x == 1 or x == n - 1:
            continue
        for _ in range(r - 1):
            x = pow(x, 2, n)
            if x == n - 1:
                break
        else:
            return False
    return True
```

### Пример 2: Рандомизирано търсене на максимален елемент
```python
import random

def randomized_max(arr, trials=10):
    max_val = arr[0]
    for _ in range(trials):
        candidate = random.choice(arr)
        if candidate > max_val:
            max_val = candidate
    return max_val
```

## 6. Common Pitfalls
- **Неправилно разбиране на вероятността за грешка** – не винаги е ясно, че Monte Carlo алгоритмите могат да се провалят с малка вероятност.
- **Игнориране на възпроизводимостта** – липсата на контрол върху random seed затруднява дебъгването и тестването.
- **Прекалено доверие в случайността** – случайните решения не гарантират оптимално решение, особено при малък брой итерации.
- **Пропускане на анализ на очакваното време** – важно е да се разбере, че случайното време за изпълнение може да варира значително.
- **Липса на достатъчно тестове** – случайните алгоритми изискват повече тестове, за да се уверим в стабилността им.
