## 2. Overview
Линейното търсене (Linear Search) е един от най-базовите алгоритми за намиране на елемент в списък или масив. Идеята му е проста: проверява се всеки елемент последователно, докато не се намери търсеният или се достигне краят на структурата. Този подход е универсален и не изисква предварително сортиране на данните.

В контекста на по-сложни системи, като AI или софтуерни приложения, линейното търсене често служи като отправна точка за разбиране на по-ефективни методи. Макар и да не е най-бързият, той е надежден и лесен за имплементация, което го прави ценен за малки или несортирани набори от данни.

Линейното търсене също така илюстрира фундаментални концепции като итерация, условни проверки и управление на потока в програмирането, които са ключови за всеки инженер в областта на софтуера и AI.

## 3. Key Concepts
- **Елемент (Element)** – Отделна единица данни в структурата, която търсим. Може да бъде число, символ или сложен обект.
- **Итерация (Iteration)** – Последователно преминаване през всеки елемент от списъка. Представете си разглеждането на страници в книга една по една.
- **Условна проверка (Conditional Check)** – Оценка дали текущият елемент съвпада с търсения. Подобно на въпрос „Това ли е?“ при разглеждане на всеки обект.
- **Времева сложност (Time Complexity)** – Мярка за това колко време отнема алгоритъмът спрямо размера на входа. Линейното търсене има O(n), където n е броят на елементите.
- **Намиране на първо срещане (First Occurrence)** – Линейното търсене обикновено връща индекса на първия намерен елемент, ако има повече от един.

3. **Анализирайте времевата сложност**  
   - Фокус: Разберете защо линейното търсене е O(n).  
   - Задача: Измерете времето за търсене в списъци с различна дължина.  
   - Въпроси: Как се променя времето при нарастване на n? Кога линейното търсене е непрактично?

4. **Сравнете с други алгоритми за търсене**

   - Въпроси: Кога е по-добре да използваме линейно търсене вместо бинарно?

## 5. Examples

**Пример 1: Линейно търсене в масив от цели числа**

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

numbers = [4, 2, 7, 1, 3]
print(linear_search(numbers, 7))  # Изход: 2
print(linear_search(numbers, 5))  # Изход: -1
```

**Пример 2: Търсене на низ в списък от низове**

```python
def linear_search_strings(words, target):
    for index, word in enumerate(words):
        if word == target:
            return index
    return -1

names = ["Анна", "Борис", "Виктор", "Георги"]
print(linear_search_strings(names, "Виктор"))  # Изход: 2
print(linear_search_strings(names, "Даниел"))  # Изход: -1
```

**Пример 3: Търсене на обект по атрибут**

```python
class Student:
    def __init__(self, name, id):
        self.name = name
        self.id = id

def search_student_by_id(students, target_id):
    for i, student in enumerate(students):
        if student.id == target_id:
            return i
    return -1

students = [Student("Иван", 101), Student("Мария", 102)]
print(search_student_by_id(students, 102))  # Изход: 1
```

## 6. Common Pitfalls
- **Прекратяване твърде рано** – Ако не се върнете с правилния резултат при липса на елемент, може да се върне грешен индекс. Винаги връщайте специална стойност като -1.
- **Игнориране на празни списъци** – Търсенето в празна структура трябва да бъде обработено, за да не доведе до грешки.
- **Липса на обработка на дублирани елементи** – Линейното търсене намира първото срещане, но ако са необходими всички, трябва да се модифицира алгоритъмът.
- **Неправилна индексация** – Внимавайте с индекси, особено при езикови особености (например 0-базирана индексация).
- **Използване при големи несортирани масиви** – Линейното търсене е неефективно при големи обеми данни, където по-оптимални методи са препоръчителни.

## 8. Quick Recap
- Линейното търсене проверява всеки елемент последователно, докато намери търсения.
- Работи при всякакви структури от данни, без нужда от сортиране.
- Времевата му сложност е O(n), което го прави неефективен при големи обеми.
- Връща индекса на първото срещане или -1, ако няма съвпадение.
- Основен алгоритъм за разбиране на търсене и итерация.
- Често използван като базова отправна точка преди по-сложни методи.
- Важно е да се обработват празни списъци и дублирани елементи правилно.
